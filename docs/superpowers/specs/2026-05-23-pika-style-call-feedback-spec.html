Pika 式音视频辩论陪练主体验

本规格把当前 WSC AI Avatar 的 `/debate` 页面收敛为一个可落地的通话训练技能： 沉浸式 Live2D 视频通话为主，训练数据以轻量启发式指标准实时出现，详细控制与记录进入自适应抽屉。

方案 1：轻量启发式反馈 不增加 LLM 旁路调用 桌面侧栏 / 移动抽屉 面向 WSC Team Debate

产品目标

让学生进入页面后感觉自己正在和一个 AI 辩手视频通话，而不是在操作一个聊天工具。 训练反馈必须准实时、低干扰、可行动：提醒学生现在该补结构、证据、时长或 POI 回应。

非目标

本轮不做句段级 AI 评分、不做完整多人 3v3 编排、不做真人摄像头上屏识别。 本轮先把现有 Live2D、STT、SSE、TTS 和 IndexedDB 能力组织成稳定主体验。

1. 通话画面优先

Live2D 舞台占据主要视觉面积。顶部只保留 call duration、motion、role/round 等短信息。 底部是圆形通话控制，贴近 Pika 的视频通话语言。

2. 训练指标轻量化

准实时反馈只使用前端 transcript 启发式规则，不阻塞 AI 回复，不消耗额外 token。 指标用小条带呈现，避免像考试成绩一样压迫学生。

3. 控制进入抽屉

Motion picker、side、round、history、provider、transcript 和 POI 放在桌面右侧窄面板； 移动端折叠为底部抽屉。

启发式反馈规则

指标 输入 状态规则 用户文案
Time 录音开始时间、当前 round Opening/Rebuttal 目标 3:30-4:00；Reply 目标 1:30-2:00。低于 30% 为 quiet，高于 90% 为 wrap up。 Good pace / Wrap up soon
Structure transcript 文本 检测 signpost：firstly, secondly, my first argument, therefore, in conclusion 等。 Signpost found / Add structure
Evidence transcript 文本 检测 example/evidence anchor：for example, study, data, in 2024, according to, case of 等。 Needs example / Evidence present
POI 上一轮 AI POI、用户 transcript 如果存在 pending POI，检测用户前 45 秒是否出现 because/however/therefore 或 POI 关键词重合。 Address POI / POI answered
Clarity 词数、句长 平均句长超过 28 词提示拆句；连续 filler 词超过阈值提示 slow down。 Shorten sentences / Clear delivery

数据流

1. Mic / Text

用户说话或输入。Web Speech API 逐步更新 interim/final transcript。

2. Heuristic Engine

前端根据 transcript、round、POI 和计时器计算 training signals。

3. Call Overlay

Time / Structure / Evidence / POI 以低干扰条带显示在通话画面底部。

4. LLM Turn

用户提交后，现有 SSE 调用生成 AI 对手回应，TTS 按句播放。

5. Session Store

messages、motion、round、feedback snapshots 写入 IndexedDB，供历史复盘。

组件边界

  • VideoCallScene 负责 Pika 式通话 chrome、指标条、折叠入口。
  • TrainingDrawer 负责 motion、role、round、history、transcript、POI。
  • useTrainingSignals 负责启发式计算，纯函数优先，便于 Vitest 覆盖。
  • TrainingMetricStrip 只渲染指标，不知道 LLM 或 TTS。

验收标准

  • 桌面默认展示视频通话 + 右侧训练抽屉；可一键折叠为纯通话。
  • 移动端默认通话，训练数据通过底部抽屉访问。
  • 用户录音或输入时，Time / Structure / Evidence / POI 在 300ms 内更新。
  • 无额外 LLM 评估调用；现有 `/api/chat`、TTS、history 流程保持可用。
  • 核心启发式规则有单元测试，页面通过 typecheck。
