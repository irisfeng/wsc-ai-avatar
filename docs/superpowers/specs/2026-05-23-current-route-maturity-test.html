成熟可用的定义

Core loop 可连续练习

用户可以完成多轮发言，AI 以对方辩手回应，训练面板持续给出可解释反馈。

Voice route 准实时

当前仍是半双工：STT → LLM → 分句 TTS → Live2D 播放。先把这个链路稳定。

Asset route Live2D

角色资产、表情映射、声音、布局参数必须可回归检查，避免 UI 调整后破坏体验。

当前体验链路

1. InputChrome 麦克风或文本输入。麦克风权限和 Web Speech API 是人工验收重点。
2. Debate contextMotion、side、round、skill 和本地 learner profile 共同组成练习上下文。
3. LLM streamAI 以 opponent mode 输出回应，前端增量显示并拆句。
4. TTS + lipsync每个完整句子进入 TTS 队列，播放时驱动 Live2D 口型和表情。
5. Training memory结束后更新 checklist、POI、next drill 和 IndexedDB 本地记录。

自动验证

高频本地回归

每次 UI、avatar、训练逻辑、TTS 相关修改后运行：

npm run verify:current

覆盖 TypeScript、Vitest、dev server、/debate 页面和默认声音 TTS smoke。

Provider 深度 smoke

确认 LLM provider 和 judge endpoint 可用时运行：

npm run smoke

该命令会调用 chat、tts、score，可能受网络、模型服务和 token 成本影响。

人工验收脚本

  • 在 Chrome 打开 /debate，允许麦克风权限，点击底部麦克风后说一段 15-25 秒英文开篇。
  • AI 回复期间，输入框不可误触，字幕显示最近句子，Live2D 有口型动作，声音与 Mao 年轻活泼形象一致。
  • 点击中断按钮时，LLM stream、TTS 队列和音频播放都停止；下一次发言可以继续。
  • 切换 Proposition/Opposition、opening/rebuttal/reply 后，AI 立场和训练 checklist 语义正确。
  • 桌面端 Training panel 不遮挡 avatar 主体；移动端抽屉可打开关闭，输入栏和通话控制不重叠。
  • 完成至少 3 轮后刷新页面，历史记录可恢复，next drill 有机会根据弱项更新。

阶段验收标准

维度通过标准失败即阻断
启动npm run verify:current 全绿。TypeScript、单测、页面或 TTS smoke 任一失败。
音频一次完整语音发言可被识别并触发 AI 回复；AI 回复可听见且可中断。麦克风无法启动、TTS 无声、停止后仍继续播放。
Avatar默认 Mao 正常加载，口型有明显响应，表情映射不报错。黑屏、模型缺失、切换 avatar 后 WebGL 崩溃。
训练Checklist、POI、next drill 与用户发言有可解释关联。训练面板静态无变化，或反馈与 round/side 明显冲突。
布局1440px 桌面、1280px 桌面、390px 手机宽度下无关键控件遮挡。输入区挡住主操作、Training 面板无法使用、文字溢出按钮。
资源10 分钟练习 CPU 和温度保持可接受；无后台重任务时风扇不过度持续拉满。温度持续高于 75°C 或浏览器内存持续增长明显。

继续推进前的风险门

不做真双工承诺

当前路线不是 Realtime API/WebRTC。验收口径必须写成准实时半双工。

资源优先级

本机有重任务时暂停长跑测试。超过 75°C 持续升温时只跑单测和静态检查。

先固化再扩展

角色、技能、训练数据都先进入回归测试，再考虑 3D 或新实时架构。
