Pika-style Call Feedback 实现路线图

这份 HTML 是实现计划的可视化执行面板：把准实时启发式反馈、Pika 式通话布局、训练抽屉和验证流程拆成可检查的工程任务。 Markdown 计划仍是逐行执行源，本文档用于快速理解、排期、review 和协作交接。

实现原则

不改 LLM、SSE、TTS 和 Live2D 核心链路。新增反馈系统必须是前端纯启发式，可测试、低延迟、不额外消耗 token。 UI 改造只重排 `/debate`，保留现有 history、motion、role、round 和输入能力。

主要边界

`lib/trainingSignals.ts` 负责判断；`TrainingMetricStrip` 负责通话画面中的低干扰展示； `TrainingDrawer` 负责训练控制和 transcript；`VideoCallScene` 只提供插槽，不知道训练规则。

Transcript Input

来自文字输入、mic interim/final、最近一条用户消息。

Signal Engine

计算 Time、Structure、Evidence、POI、Clarity。

Call Overlay

在 Live2D 通话画面上显示 4 个核心指标。

Training Drawer

承载 motion、side、round、history、POI、transcript。

质量门槛

无额外 LLM 成本

反馈来自 transcript 启发式规则，不新增评分 API 调用。

300ms 级 UI 更新

输入和 mic interim 变化后直接重新计算 signals。

响应式可用

桌面右侧面板，移动端底部抽屉，主通话画面不丢失。

可回滚增量

每个任务独立提交，问题可定位到单个边界。

执行建议

优先使用 Subagent-Driven：Task 1 和 Task 2 可以独立并行 review；Task 3 和 Task 4 顺序执行； Task 5 需要本地浏览器验证。若资源紧张，先完成 Task 1-3，不启动 dev server。
